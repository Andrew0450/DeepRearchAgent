```
│       │              │              │                    │
│       │              │              ↓                    │
│       │              │         ④ 筛选决策                 │
│       │              │              │                    │
│       │              │              ↓                    │
│       │              │         ⑤ 方案设计                 │
│       │              │              │                    │

│       │              │         ⑥ 成果输出                 │
```

### 6个阶段说明

| 阶段 | 输入 | 输出 | 执行者 |
|------|------|------|--------|
| ① 定义主题 | 模糊的想法/关键词 | 明确的调研问题 | 人 + AI |
| ② 选择方向 | 调研问题 | 方向模板（A/B1/B2/C/D/E） | 人（AI建议） |
| ③ 信息采集 | 关键词 + 方向配置 | 结构化数据（CSV/表格/要点） | 专业工具为主，AI辅助 |
| ④ 筛选决策 | 结构化数据 + 筛选标准 | 最优选项/核心结论 | AI为主，人确认 |
| ⑤ 方案设计 | 核心结论 + 讲稿模板 | 结构化方案/论证框架 | AI |
| ⑥ 成果输出 | 方案 + 模板 | 讲稿 + PPT提示词 + 参考文献 | AI |

### 调研管线模式

当项目需要多维度调研时，各方向可串联为管线，上游产出作为下游输入：

```
B1(行业定位-宏观) → E(政策环境) → B2(市场量化-中观) → D(竞品对比-微观) → A/C(技术深挖)
```

**管线原则**：
- 每个方向只做自己的事，不重复其他方向的内容
- 需要其他方向的内容时，用"→见方向X输出"引用，而非重写
- 可从管线任意节点开始，不必从B1开始
- **边界清晰≠内容缩减**：每个方向在自己的边界内必须搜深搜透。"→见方向X"仅指不在本方向报告中重复书写其他方向的内容，而非跳过搜索。当需要某个方向的信息但尚未执行时，仍需在本方向内做必要的搜索，只是产出时引用而非重写。

**各方向核心边界**：

| 方向 | 视角 | 核心问题 | 只做 | 不做 |
|------|------|---------|------|------|
| B1 | 宏观 | 赛道值不值得进？ | 行业结构、产业链、规模、格局、痛点 | 政策详细分析（→E）、市场细分量化（→B2） |
| E | 制度 | 政策风向是什么？ | 政策梳理、解读、影响方向、落地进展 | 产业规模量化（→B2）、竞品分析（→D） |
| B2 | 中观 | 市场多大、怎么切？ | 市场量化、应用场景、产业链价值、痛点 | 政策详细分析（→E）、竞品深度对比（→D） |
| D | 微观 | 怎么赢？ | 竞品对比、优劣势、推荐方案、迁移路径 | 市场规模（→B2）、政策解读（→E） |
| A/C | 技术 | 技术方案怎么选？ | 技术原理、方案对比、实验效果、创新点 | 行业全景（→B1）、市场价值（→B2） |

> 参考框架：行业分析(宏观) → 市场分析(中观) → 竞品分析(微观)，三个层级视角不同、工具不同、产出不同。

## 二、工作流铁律

> 所有调研工作必须遵循以下铁律，确保产出质量可追溯、可验证、可复现。

### 铁律一：工具有序

不同类型的信息采集应使用最专业的工具：

| 信息类型 | 推荐工具 | 说明 |
|---------|---------|------|
| 学术论文 | Consensus / Semantic Scholar | 专业论文检索，质量排序 |
| 行业数据 | 艾瑞/亿欧/头豹/证券研报 | 权威行业报告 |
| 政策文件 | 国务院/各部委官网 | 原文优先，避免二手解读 |
| 竞品分析 | 产品官网 / GitHub | 第一手信息 |
| 全球统计 | Statista / Euromonitor / 世界银行 | 国际权威数据 |

**核心原则**：专业工具做专业事，AI做整合。不要用AI搜索替代专业检索工具。

### 铁律二：来源可溯

每条关键数据/论点必须有来源，格式为：
- **正文中提及数据时，直接内联附超链接**（如"据[艾瑞咨询2025年报告](URL)..."），无需跳到附录查找
- 附录中同样列出完整来源表，**每条来源必须附可点击的超链接URL**
- 数据来源网站如QYResearch、Statista等，必须附具体报告页面链接
- 没有来源的数据标注"来源待验证"

**来源标注格式**：`[来源名称](URL)`

**核心原则**：读者在正文看到数据时，点击即可溯源，不需要翻到附录去找链接。

**示例**：
> 据[QYResearch 2024年全球报告](https://www.qyresearch.com.cn)，中国无损检测设备市场规模约4.54亿美元。

### 铁律三：数据有时效

| 数据类型 | 有效期 | 超过有效期处理 |
|---------|-------|--------------|
| 市场规模/增速 | 2年内 | 需标注"数据截至XX年"，或更新数据 |
| 政策文件 | 长期有效 | 标注发布/生效时间，关注修订 |
| 技术参数 | 1年内 | 标注数据时间，警惕技术迭代 |
| 用户评价 | 实时 | 关注最新评价，警惕刷评 |

**核心原则**：优先使用最新数据，超过2年的数据必须显式标注。

### 铁律四：估算透明

当某个关键数据无法通过直接搜索获得，但可以通过已有数据合理推算时：

**允许估算的条件：**
- 至少有2个可靠的关联数据点作为估算基础
- 估算逻辑清晰、可复现

**估算必须包含：**
1. 估算公式（如：市场B = 市场A × 渗透率）
2. 每个参数的来源（如：市场A据QYResearch 2024年为4.54亿美元）
3. 估算的假设条件（如：假设渗透率从15%增长到25%）
4. 标注"估算值"（与实际数据明确区分）

**格式示例：**
> 中国水果无损检测设备市场规模 ≈ 全球市场4.54亿美元 × 中国份额28% ≈ 1.27亿美元（约9.1亿元）
> - 全球市场：QYResearch 2024年数据
> - 中国份额：产业世界2024年数据
> - ⚠️ 此为估算值，非官方数据

## 四、经验更新机制

> 借鉴 #Agent成长追踪 的自动蒸馏机制，与现有"经验萃取"思路互补

### 决策过程库更新链路

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1. 模式发现 | 从踩坑记录中发现重复模式 | 关键任务完成后自动触发 |
| 2. 准则生成 | 转化为结构化准则条目 | 如"⚠️ fetch_web无法读PDF，需云电脑" |
| 3. 去重检查 | 对比已有准则重叠度 | 避免重复覆盖 |
| 4. 审批入库 | 明确确认后才正式入库 | 质量把控 |

### 遗忘曲线机制

- **降级策略**：长期未触发的经验自动标注为"⚠️待验证"
- **定期清理**：每月扫描决策过程库，清理过时或被推翻的条目
- **保持精简**：经验库应"活而不杂"，避免成为垃圾场

### 迭代优化流程
> 借鉴"技能优化大师"的自迭代机制，适用于核心文件优化

| 轮次 | 操作 | 产出 |
|------|------|------|
| 第1轮 | 初次沉淀 → 自评测(附A) | 发现问题清单(P0/P1/P2) |
| 第2轮 | 针对P0/P1优化 → 重评 | 验证修复效果 |
| 第3轮 | 细节打磨 → 达标 | 正式入库 |

**停止条件**：
- 达到"良好"(70分)以上 → 正式入库
- 3轮后提升<5分 → 停止迭代，接受当前水平

**适用场景**：TOOLS.md / SKILL.md / SOP README.md 等核心文件的优化

### 沉淀层级

| 内容类型 | 沉淀位置 | 更新频率 |
|---------|---------|---------|
| 工具使用技巧 | TOOLS.md | 按需 |
| 决策推理过程 | TOOLS.md 决策过程库 | 关键任务后 |
| 调研方法论 | SOP 铁律/模板 | 重大变更需确认 |
| 身份/规范类 | SOUL.md/USER.md | 极低，需用户确认 |

## 五、方向模板

### 方向 A：技术方案调研

**适用场景**：课设/毕设/项目，需要找到可复现的技术方案

**典型输入**：基于XXX的YYY系统设计 / XXX技术在ZZZ场景的应用

#### 信息采集配置

| 信源 | 采集方式 | 优先级 | 官网URL |
|------|---------|--------|---------|
| Consensus (Pro模式) | 自然语言提问 → 下载CSV | ★★★★★ | https://consensus.app/ |
| Semantic Scholar API | API检索 / 网页搜索 | ★★★★ | https://www.semanticscholar.org/ |
| 秘塔AI搜索 | 研究模式/学术搜索 | ★★★★ | https://metaso.cn/ |
| Google Scholar | 关键词搜索 → 摘要阅读 | ★★★ | https://scholar.google.com/ |
| 知网 (AI模式) | AI搜索 → 参考文献列表 | ★★★★ | https://cnki.net/ |
| arXiv | 关键词搜索 | ★★★ | https://arxiv.org/ |
| arXiv论文助手（虾评技能） | 免费API搜索+AI摘要+关键词+引用数 | ★★★★ | https://xiaping.coze.com/skill/e8b02dd2-5519-4b5a-9ca1-d5d618181b6 |

**采集要点**：
- 不需要阅读全文，只拿结构化数据（CSV/表格/参考文献列表）
- 如果用Consensus：Pro模式提问 → 直接下载CSV，不需要看Pro Analysis的结果
- 如果用Google Scholar：快速扫摘要，标记相关论文到列表
- 如果用知网：AI模式搜索，直接导出参考文献
- 如果用arXiv论文助手：免费API（每日1万次请求，响应约50ms），可获取AI摘要(tldr)、关键词、引用数、全文Markdown，适合快速筛选

#### 筛选标准

| 维度 | 权重 | 说明 |
|------|------|------|
| 创新性 | 30% | 方案有明确创新点，不是纯复现 |
| 可复现性 | 35% | 有明确的实验/仿真方案，数据可得 |
| 难度适中性 | 25% | 适合大三学生水平，不需要超算/特殊硬件 |
| 时效性 | 10% | 近3-5年论文优先 |

**筛选方法**：
1. 将CSV喂给AI（Gemini/扣子），要求按上述维度评分
2. AI输出Top 3-5篇推荐论文，每篇附评分和推荐理由
3. 人确认最终选择的1篇论文

#### 讲稿结构（模板A1）

```
1. 技术背景（精简，2-3句话点题）
   - 技术领域的核心问题
   - → 详细行业背景见方向B1输出（如尚未执行方向B1，本方向仍需搜索相关行业信息，只是在产出中标注归属方向B1）
2. 是什么
   - 技术/方案的核心定义
   - 关键原理简述
3. 技术发展趋势
   - 技术演进方向
   - 学术/产业前沿
4. 方案对比
   - 主流技术方案横向对比（表格）
   - 各方案优劣势分析
5. 技术痛点
   - 现有技术方案的核心瓶颈
   - 未解决的技术挑战
   - → 行业层面痛点见方向B1/B2输出（如尚未执行方向B1/B2，本方向仍需搜索相关行业痛点信息，只是在产出中标注归属方向B1/B2）
6. 我们的方案
   - 选定论文的核心方法
   - 技术路线与架构
7. 效果
   - 论文中的实验结果
   - 关键指标对比
8. 创新点
   - 与现有方案的区别
   - 技术贡献
9. 应用前景（精简）
   - 核心应用场景
   - → 市场价值详细分析见方向B2输出（如尚未执行方向B2，本方向仍需搜索相关市场价值信息，只是在产出中标注归属方向B2）
```

> **A1边界**：聚焦技术层面（技术原理、方案对比、实验效果、创新点）。行业/市场/政策背景引用其他方向输出，但搜索时仍需覆盖这些信息。

#### 讲稿结构（模板A2）

```
1. 技术背景（精简，2-3句话点题）
   - 技术领域的核心问题
   - → 详细行业背景见方向B1输出（如尚未执行方向B1，本方向仍需搜索相关行业信息，只是在产出中标注归属方向B1）

   - 现有技术方案的核心瓶颈
   - → 行业层面痛点见方向B1/B2输出（如尚未执行方向B1/B2，本方向仍需搜索相关行业痛点信息，只是在产出中标注归属方向B1/B2）
3. 我们的方案
4. 创新点
5. 效果对比
6. 项目成果
7. 应用价值（精简）
   - 核心应用场景
   - → 市场价值详细分析见方向B2输出（如尚未执行方向B2，本方向仍需搜索相关市场价值信息，只是在产出中标注归属方向B2）
```

> **A2边界**：聚焦技术层面（技术原理、方案对比、实验效果、创新点）。A2偏应用展示，适合项目汇报场景。行业/市场/政策背景引用其他方向输出，但搜索时仍需覆盖这些信息。

**市场规模测算方法**：

| 方法 | 逻辑 | 适用场景 | 示例 |
|------|------|---------|------|
| 自上而下 | 大市场 × 细分比例 | 有宏观数据 | 中国水果市场 × 检测设备渗透率 |
| 自下而上 | 用户数 × 客单价 × 频次 | 细分维度清晰 | 种植户数量 × 设备单价 × 采购率 |
| 类比法 | 参照成熟市场推算 | 新兴市场、缺数据 | 参照日本水果检测设备渗透率 × 中国市场 |

**TAM/SAM/SOM三层量化**：
- TAM（总潜在市场）= 目标用户基数 × 平均客单价 × 年均消费频次
- SAM（可服务市场）= TAM × 可触达用户比例
- SOM（可获得市场）= SAM × 预期市场占有率

> 估算时按铁律四执行：必须包含公式、参数来源、假设条件，标注"估算值"。

### 流程分层设计（借鉴 data-analysis-flow）

> 借鉴 #虾评 data-analysis-flow 的分层架构思想，将调研流程分层执行

参考其七层架构（L1人设理解 → L2数据范围获取 → L3分析数据范围 → L4问题拆解 → L5方法选择 → L6执行计算 → L7结果输出），调研流程可分为：

| 层级 | 任务 | 说明 |
|------|------|------|
| L1 | 理解需求 | 明确调研问题、输出形式 |
| L2 | 信源配置 | 确定采集工具和关键词 |
| L3 | 信息采集 | 执行搜索，获取原始数据 |
| L4 | 筛选决策 | 去重、评估、选择 |
| L5 | 分析整合 | 结构化处理、对比分析 |
| L6 | 方案设计 | 结论提炼、建议生成 |
| L7 | 成果输出 | 报告生成、格式优化 |

**分层原则**：
- 每层只做自己的事，不跳跃也不越界
- 上一层输出作为下一层输入，链路清晰
- 适配多方向管线模式，上下游边界明确

**行业适配指南**：

不同行业的竞品分析侧重点不同，执行时按行业调整对比维度和信源：

| 行业 | 核心对比维度 | 关键信源补充 |
|------|------------|------------|
| 科技/互联网 | 功能覆盖、API生态、开发者社区、迭代速度 | GitHub、Product Hunt、技术博客 |
| 消费品/零售 | 定价策略、渠道覆盖、品牌认知、用户口碑 | 电商数据、社交媒体、用户评论 |

## 四、执行流程详解

### Step 1：定义主题

**输入**：模糊的想法、一个关键词、或者一个具体问题

**操作**：
1. 将模糊想法转化为明确的调研问题
2. 如果主题太宽，拆分为子问题
3. 确定调研的时间范围和深度要求

**产出**：1-3个明确的调研问题

**示例**：
- 模糊输入：「我想做无人机的路径规划」
- 转化后：「在消费级无人机场景下，基于强化学习的动态路径规划方法有哪些？哪些方案适合在仿真环境中复现？」

### Step 2：选择方向

**操作**：
1. 根据调研目的，从A/B1/B2/C/D/E中选择方向
2. 如果目的不明确，用以下判断逻辑：

```
你的目的是什么？
├── 找到可复现的技术方案 → 方向A
├── 了解一个行业的全貌 → 方向B1（行业驱动型）
├── 了解一个产品背后的行业 → 方向B2（产品驱动型）
├── 写综述/找研究空白 → 方向C
├── 对比同类产品/方案 → 方向D
└── 分析政策/趋势 → 方向E

注意：
- B1和B2的调研逻辑完全不同，B2需要从产品向外扩展三层
- 如果既要B2又要D，可以并行执行：B2做行业调研，D做竞品分析，最后合并
```

3. 选择方向后，确认使用的讲稿模板

**产出**：方向编号 + 讲稿模板选择

### Step 3：信息采集

**核心原则**：专业工具做专业事，AI做整合

**操作**：

#### 3.1 论文类采集（方向A/C）

```
方式1（推荐）：Consensus
  → 打开 consensus.app
  → 用自然语言提问（Pro模式）
  → 下载CSV（不需要看Pro Analysis结果内容）
  → CSV交给AI处理

方式2：Semantic Scholar
  → 通过API或网页搜索
  → 可让扣子直接调API获取结构化数据
  → 免费且覆盖面接近Consensus

方式3：Google Scholar
  → 关键词搜索
  → 按被引量排序
  → 快速阅读摘要，标记相关论文

方式4：知网（中文论文）
  → 使用AI模式搜索
  → 导出参考文献列表
```

#### 3.2 行业数据类采集（方向B1/B2/E）

```
→ 搜索行业报告（艾瑞/亿欧/头豹/证券研究所）
→ 搜索政策文件原文
→ 搜索上市公司年报/招股书
→ 可让扣子通过联网搜索辅助采集
→ 关键数据标记来源和年份
```

#### 3.3 产品/竞品类采集（方向D）

```
→ 收集产品官网信息
→ 检查GitHub仓库（Star/Fork/Issue）
→ 搜索技术评测/对比文章
→ 建立对比维度表
```

### 3.4 图片数据提取

研报和行业报告中，很多关键数据以图表形式呈现（如市场规模柱状图、份额饼图等），无法直接复制文字。

**处理方法：**
1. 搜索到含数据图表的文章/研报时，提取图片
2. 使用图片内容分析工具（OCR）识别图表中的数据
3. 识别后在报告中标注："数据来源于XX报告图表，经AI识别提取"
4. 同时附上原图链接，方便人工核验

**适用场景：**
- 微信公众号中的图片版研报转载
- PDF研报中的图表
- 网页中的数据可视化图表

### 3.5 微信公众号搜索深度策略

微信公众号是获取以下内容的重要渠道：
1. **图片版研报转载**：付费研报经常被以图片形式完整转载（虽然盗版但内容完整）
2. **消费者调研文章**：如水果购买偏好、消费趋势等
3. **行业观察与深度分析**：从业者视角的市场洞察

**搜索方法：**
- 搜狗微信搜索：https://weixin.sogou.com/（主要入口）
- 微信内搜索：搜索框→公众号类型筛选
- 关键词策略："XX行业 研报""XX 深度报告""XX 白皮书""XX 消费趋势"

**获取文章内容：**
- 搜索到微信文章URL后，使用网页读取工具获取全文
- 对于图片版研报，使用图片识别工具提取数据
- 微信文章URL格式：https://mp.weixin.qq.com/s/XXXXX

**可信度分级：**
- 图片版原文转载：★★★★（内容完整，接近原版）
- 文字摘要+分析：★★★（可能有删减或观点偏差）
- 二次创作/解读类：★★（原创性高但可信度下降）

**⚠️ 注意**：作为参考可以，但正式引用最好能找到原出处。

**产出**：结构化数据文件（CSV/表格/要点列表）

### Step 4：筛选决策

**操作**：
1. 将采集数据 + 方向筛选标准 喂给AI
2. AI按各维度的权重进行评分
3. AI输出推荐清单（Top 3-5），每项附评分和理由
4. **人确认最终选择**

**产出**：1个核心选项 + 选择理由

### Step 5：方案设计

**操作**：
1. 基于选定选项，按照对应方向的讲稿模板展开
2. 紧密结合源材料（论文/报告/数据），不是空对空写
3. 遵循"发现问题 → 解决问题"的逻辑主线
4. 每个论点有数据或引用支撑

**产出**：结构化的方案框架（Markdown大纲）

### Step 6：成果输出

**操作**：
1. 基于方案框架，生成完整讲稿（Markdown全文）
2. 生成PPT提示词（结构化prompt，包含每页的标题、要点、布局建议）
3. 整理参考文献/数据来源列表

**产出**：
- ✅ 讲稿全文（.md）
- ✅ PPT提示词（可直接喂给WPS AI / 扣子）
- ✅ 参考文献列表

## 七、常见问题

### Q：Consensus和我的搜索差距有多大？

Consensus底层是Semantic Scholar（2.2亿+论文）+ OpenAlex + 自有爬取，三步检索（语义+关键词→质量排序→精排）。我的搜索能覆盖Google Scholar和Semantic Scholar的大部分内容，但检索精度和排序算法不如Consensus专业。

**建议**：正式课设用Consensus采集，日常快速调研可以让我全托管。

### Q：知网论文怎么处理？

目前我无法直接访问知网。建议你用知网AI模式搜索后，把参考文献列表截图或复制文本喂给我，我来处理后续环节。

### Q：可以组合多个方向吗？

可以。比如"行业调研+技术方案"可以拆成两轮：先用方向B1了解行业全貌，再用方向A深挖技术方案。"产品调研+竞品分析"可以并行执行：方向B2做行业调研，方向D做竞品分析，最后合并成完整报告。

### Q：讲稿结构可以自定义吗？

完全可以。上面提供的模板是默认方案，你可以自由调整章节顺序、增减内容。告诉我你的需求，我按你的结构来。

### Q：PPT生成支持什么方式？

1. 我生成结构化PPT提示词，你喂给WPS AI生成
2. 我直接用create-ppt技能生成PPT文件（.pptx.html格式）
3. 两种方式都可以，看你的偏好

### Q：微信公众号转载的研报可靠吗？

微信公众号是获取专业研报的重要渠道，很多付费研报会以图片形式完整转载。判断可靠性要看具体形式：
- **图片版研报转载**（原文截图/扫描件）：可信度较高，接近原版，内容完整
- **二次创作的解读类推文**：可信度会打折扣，可能存在删改、提炼偏差
- AI可以识别图片中的文字内容，所以图片版研报也能提取数据

建议：可以作为快速了解行业的参考，正式引用最好能找到原出处（如券商官网、慧博投研等）。我会在来源附录中标注"转载，需核实原文"。

### Q：冷门竞品怎么找？

采用分层搜索策略——先找头部产品，再通过垂直渠道（GitHub/Product Hunt/众筹平台）找细分产品，最后用长尾关键词+多语言搜索深挖冷门产品。电商平台（淘宝/京东/Amazon）的搜索和筛选功能也能帮助发现竞品。

### Q：秘塔AI搜索的API怎么接入？

访问 metaso.cn 注册账号，在 https://metaso.cn/search-api/api-keys 获取API Key，然后提供给我。我会在工作流中集成秘塔搜索，特别是中文研报和行业数据的搜索场景。

### Q：电商平台上的信息可靠吗？

产品参数和定价信息来自官方，可靠性高（★★★★★）；用户评价需要甄别刷评，可靠性中等（★★★）；销量数据仅供参考，不同平台统计口径不同。

### Q：B1和B2有什么区别？

| 维度 | B1（行业驱动型） | B2（产品驱动型） |
|------|------------------|------------------|
| 起点 | 一个行业名称 | 一个具体产品 |
| 逻辑 | 从行业向下拆解 | 从产品向外扩展 |
| 典型输入 | "人形机器人产业调研" | "水果无损检测仪" |
| 覆盖范围 | 产业链、市场、玩家、趋势 | 产品市场→应用场景→宏观环境 |
| 讲稿模板 | B1 | B2 |

## 八、信息来源可靠性规范

### 附录规范

所有调研产出必须包含来源附录，格式如下：

```
## 附录：信息来源

| 序号 | 内容 | 来源 | URL | 可靠性评级 |
|------|------|------|-----|-----------|
| 1 | xxx | xxx | [链接文本](URL) | ★★★★★ |
```

> **重要**：每条来源必须附可点击的超链接URL，使用 `[链接文本](URL)` 格式。

### 可靠性评级标准

| 评级 | 含义 | 适用场景 |
|------|------|---------|
| ★★★★★ | 官方原文/权威机构一手数据 | 核心论点、政策文件、技术规格 |
| ★★★★ | 权威媒体/知名研究机构 | 行业数据、研报摘要、新闻事件 |
| ★★★ | 第三方平台/社区讨论 | 用户评价、论坛讨论、评测文章 |
| ★★ | 转载/非官方渠道（需核实原文） | 微信公众号转载内容 |
| ★ | 未验证/来源不明 | AI推测、无法追溯的数据 |

### 核心原则

1. **每条关键数据/论点必须有来源**：调研报告中的市场规模、增速、份额等数据，必须标注来源
2. **所有来源必须附超链接**：使用 `[来源名称](URL)` 格式，方便点击溯源
3. **微信公众号转载内容需标注**：在备注栏标注"转载，需核实原文"
4. **无法追溯原文的数据降级处理**：如果无法找到原始出处，将评级降低并说明
5. **AI生成的综合分析需标注**：在备注栏标注"AI综合分析，建议交叉验证"
6. **优先使用一手来源**：能找到官方原文就不看二手解读

### 来源优先级

```
官方原文 > 权威机构 > 专业媒体 > 行业平台 > 社区讨论 > AI综合
```
